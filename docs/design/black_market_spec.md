# Black Market — 세부 기획서

프로토타입 가상 거래소. 실제 Steam 거래 없이 "시장에 올려두면 팔린다"는 체감을
제공하고, 다른 유저가 올린 듯한 매물을 구매할 수 있게 한다. 시세는 품목별로
출렁이며 세이브에 유지된다.

---

## 0. 확정된 설계 결정

| 항목 | 결정 |
|------|------|
| 오프라인 판매 | ❌ 접속 중에만 판매 진행 |
| 판매 체감 속도 | 분 단위 압축 (급매 0~0.5분 ~ 비싸면 미판매, 최대 5분) |
| 시세 변동 | 품목별 시세 지수가 출렁임 |
| 시세 영속성 | 세이브에 저장·유지 |
| 통화 | 기존 루블 단일 |
| 진입 동선 | 메인 화면 상단 탭/버튼 (`MARKET`) |
| 동시 매물 수 | 5개 |
| 등록 수수료 | 등록가의 5%, 등록 시 선지불(반환돼도 소모) |

## 1. 시세 시스템 (Market Index)

### 1.1 개념
품목(`item_id`)마다 **시세 지수** `index ∈ [0.6, 1.6]`를 둔다. 기준값 1.0.
실제 시장가 = `base_value × index`.

> `base_value`는 아이템 정의(JSON)의 기준가. 품질 배율은 **드롭 인스턴스**에
> 이미 반영돼 있으므로, 시세는 "그 품목 카테고리의 인기/시세"를 표현한다.
> 즉 시세 지수는 **품목 id 단위**로 적용하고, 개별 인스턴스의 품질 프리미엄은
> 인스턴스의 `base_value`(품질 반영가)에 그대로 곱한다.

### 1.2 변동 로직 (랜덤 워크 + 평균 회귀)
브라우즈 갱신 주기(5분, §2.1)마다 모든 등장 품목의 지수를 갱신:

```
drift   = randf_range(-0.08, 0.08)          # 무작위 변동
revert  = (1.0 - index) * 0.10              # 1.0 쪽으로 약하게 회귀
index   = clampf(index + drift + revert, 0.6, 1.6)
```

- 평균 회귀로 지수가 한쪽으로 영구히 쏠리지 않음.
- 한 번 갱신 시 최대 ±8%p 이동 → "어제보다 시세 올랐다" 체감.

### 1.3 영속성
- `game_state.market.index = { item_id: float }` 형태로 세이브에 저장.
- 세이브에 없는 품목은 최초 등장 시 1.0으로 초기화.
- 세션을 넘어 시세 흐름이 이어진다 (장기 추적 가능).

## 2. 브라우즈 탭 (구매)

### 2.1 매물 생성
- **5분(300초)마다** 매물 풀 전체 재생성 (접속 중 실시간).
- 1회 생성 매물 수: **6~8개** (randi_range).
- 매물은 세이브하지 않음(휘발). 재접속/갱신 시 새로 생성.

### 2.2 등장 품목 — 플레이어 레벨 가중
| 레벨 | 티어 분포 |
|------|----------|
| Lv.1–2 | T1 70% / T2 30% |
| Lv.3–5 | T1 20% / T2 60% / T3 20% |
| Lv.6–9 | T2 25% / T3 65% / T4 10% |
| Lv.10+ | T3 50% / T4 50% |

> Named Legendary 아이템은 **이번 범위 밖**(미구현). Lv.10+ Named 출현은 Layer 2
> 구현 후 별도 추가.

품목 선정: 해당 티어 풀에서 무작위 1개. 품질은 브라우즈 매물에도 적용
(드롭과 동일한 분포로 롤하여 매물 인스턴스 생성).

### 2.3 가격 책정
각 매물 가격 = `instance.base_value × index(item_id) × variance`

| 구간 | 확률 | variance | 비고 |
|------|------|----------|------|
| 일반 | 90% | 0.85 ~ 1.20 | 시장가 근처 |
| 프리미엄 | 8% | 1.30 ~ 2.00 | 비싼 매물 |
| 급처(Fire-sale) | 2% | 0.50 ~ 0.75 | 즉시 구매 유도 |

급처 매물은 시각 강조(예: 빨간 `급처` 배지) + 다음 갱신(5분) 전에도
**30초 후 사라짐**.

### 2.4 구매 동작
- 루블 충분 시 `BUY` 활성. 부족하면 비활성.
- 구매: `rubles -= price`, 매물 인스턴스를 `inventory`에 추가.
- 구매 즉시 해당 매물은 풀에서 제거.
- `rubles_changed`, `inventory_changed` emit.

## 3. 내 매물 탭 (판매 시뮬레이션)

### 3.1 등록
- 동시 최대 **5개**.
- **등록 동선은 거래소 내부에서 완결**: `MARKET → MY LISTINGS` 탭의
  "등록 가능한 아이템"(인벤토리 모드 전체) 목록에서 `등록` → 희망가 입력
  (기본값 = `base_value × index` 제시). 인벤토리 패널에는 등록 버튼을 두지 않음
  (장르 관습: Flea Market / 경매장은 거래소 화면에서 등록).
- **등록 수수료 = 희망가의 5%**, 등록 즉시 루블 차감.
  - 판매 실패로 반환돼도 수수료는 **돌려주지 않음**(스팸 방지·루블 소비처).
- 등록 시 아이템은 인벤토리에서 빠지고 매물로 이동(중복 판매 방지).
- 슬롯이 가득(5/5)이면 "등록 가능한 아이템"의 `등록` 버튼 비활성화 + 안내.

### 3.2 판매 시간 — 분 단위 압축
희망가 비율 `r = 희망가 / (base_value × index)` 기준, **접속 누적 시간**으로 판정:

| r | 판매 소요(접속 시간) |
|------|----------|
| ≤ 0.75 | 0 ~ 0.5분 (급매) |
| 0.75 ~ 0.95 | 0.5 ~ 1.5분 |
| 0.95 ~ 1.10 | 1.5 ~ 3분 |
| 1.10 ~ 1.30 | 3 ~ 5분 |
| > 1.30 | 미판매 (만료 후 반환) |

> 전 구간이 매물 최대 만료(5분) 안에 들어오도록 압축. `r > 1.30`만 미판매로 만료.

- 등록 시 위 구간에서 `sell_at`(접속 누적 초)를 한 번 정해 저장.
- 매물의 진행 시간은 **접속 중에만** 누적(`elapsed_online`).
- `elapsed_online ≥ sell_at` 도달 시 판매 체결.

### 3.3 판매 체결
- `rubles += 희망가` (수수료는 등록 시 이미 지불).
- 매물 제거, `listing_sold(item, price)` emit → "판매 완료" 알림 표시.

### 3.4 만료
- `r > 1.30` 또는 등록 후 **5분(접속 시간)** 경과 시 미판매 만료.
- 아이템을 인벤토리로 반환. 수수료 미반환.

## 4. 데이터 구조 / 영속성

### 4.1 game_state 확장
```gdscript
"market": {
    "index": {},        # { item_id: float }  시세 지수, 영속
    "listings": [],     # 내 매물(아래 구조), 영속
}
```

매물(listing) 구조:
```gdscript
{
    "item": { ... },            # 아이템 인스턴스 전체
    "asking_price": 6480,       # 희망가
    "elapsed_online": 0.0,      # 접속 누적 진행 초
    "sell_at": 240.0,           # 판매 체결 목표 초 (-1 = 미판매)
    "expire_at": 300.0,         # 만료 시각(접속 초)
}
```

### 4.2 저장/로드
- `index`, `listings`는 세이브에 포함(`game_state` 안에 있으므로 자동).
- 브라우즈 풀(`MarketSystem`의 휘발 데이터)은 저장하지 않음.
- 마이그레이션: 구 세이브에 `market` 없으면 `_load_save()`에서
  `{"index": {}, "listings": []}`로 초기화(기존 ammo/weapon 마이그레이션과 동일 패턴).

## 5. 신규 시스템: MarketSystem (autoload)

`AmmoSystem`과 동일하게 **자체 타이머**로 구동(레이드와 무관, 접속 중 항상).

### 5.1 책임
- 브라우즈 매물 풀 보유·5분마다 재생성.
- 시세 지수 갱신(브라우즈 갱신과 동기).
- 1초 틱: 내 매물 `elapsed_online` 누적 → 판매/만료 체결.
- 급처 매물 30초 만료 처리.

### 5.2 시그널
```gdscript
signal market_refreshed()                       # 브라우즈 풀 갱신됨
signal listing_sold(item: Dictionary, price: int)
signal listings_changed()                       # 내 매물 추가/제거/체결
```

### 5.3 주요 API
```gdscript
func get_browse_listings() -> Array
func get_market_price(item: Dictionary) -> int           # base_value × index
func buy_listing(listing_index: int) -> bool
func list_item(item: Dictionary, asking_price: int) -> bool   # 수수료 차감 포함
func cancel_listing(listing_index: int) -> bool          # 즉시 회수(수수료 미반환)
func get_my_listings() -> Array
```

### 5.4 타이머 패턴
```gdscript
func _ready() -> void:
    _tick_timer = Timer.new(); _tick_timer.wait_time = 1.0
    _tick_timer.timeout.connect(_on_tick); add_child(_tick_timer); _tick_timer.start()
    _refresh_browse()   # 최초 1회

func _on_tick() -> void:
    _refresh_accum += 1.0
    if _refresh_accum >= 300.0:
        _refresh_accum = 0.0
        _drift_index(); _refresh_browse(); emit market_refreshed
    _advance_listings(1.0)   # 판매/만료 체결
```

## 6. UI (MainUI 확장)

### 6.1 진입
- 메인 상단 버튼 행에 `MARKET` 버튼 추가(기존 워크샵/총기 버튼과 동일 스타일).
- 누르면 `_market_panel` 오버레이 표시(기존 결과 팝업/건모드 패널과 동일 패턴).

### 6.2 패널 레이아웃
```
┌─────────────────────────────────────────┐
│ BLACK MARKET              ₽ 12,440   [X] │
├─────────────────────────────────────────┤
│  [BROWSE]   [MY LISTINGS (2/5)]          │
├─────────────────────────────────────────┤
│ (BROWSE) 다음 갱신 04:12                  │
│ [T3·순정] AK DTK-4 Supp  ₽6,480   BUY    │
│ [T2] M4 Strike Comp      ₽2,100   BUY    │
│ [T1·급처] Iron Sights      ₽480   BUY    │
│ ...                                       │
├─────────────────────────────────────────┤
│ (MY LISTINGS)                             │
│ ─ 등록된 매물 ─                            │
│ Holosun HS510C  ₽2,200  판매중 03:40 [취소]│
│ AK 30rd Mag     ₽1,100  급매중 00:50 [취소]│
│ ─ 등록 가능한 아이템 ─                     │
│ M4 Strike Comp  [T2]      시세 ₽2,100 [등록]│
│ Zenitco Grip    [T3·순정] 시세 ₽3,400 [등록]│
└─────────────────────────────────────────┘
```

### 6.3 표시 규칙
- 품질 배지: 기존 `_quality_label`/`_quality_color` 재사용.
- 급처 매물: 빨간 `급처` 배지 + 잔여 시간.
- 내 매물: 남은 예상 판매 시간(`sell_at - elapsed_online`) 카운트다운, `취소` 버튼.
- 판매 완료 시 토스트/로그 한 줄(예: "✔ Holosun HS510C 판매 ₽2,200").
- MY LISTINGS는 2단 구성: `─ 등록된 매물 ─`(활성 매물) + `─ 등록 가능한 아이템 ─`
  (인벤토리 모드 전체, 행마다 티어·품질 배지 + 현재 시세 + `등록` 버튼).
- 카운트다운은 1초 타이머가 **텍스트만 in-place 갱신**(행 전체 재생성 X) →
  스크롤 유지·클릭 유실 방지. 행 재구성은 데이터 변경 시그널로만.

### 6.4 등록 플로우
- `MY LISTINGS` 탭의 "등록 가능한 아이템"에서 `등록` → 희망가 입력 다이얼로그.
  (프로토타입 단순화: `SpinBox`로 기본가 제시 + step 조정)
- 등록 시 수수료 안내("등록 수수료 ₽324 차감") 후 확정.
- 슬롯 5/5면 `등록` 버튼 비활성화.
